# Зміни в проєкті TaskSprout

## Дата: 05.07.2026

### Проблема
Веб-версія додатку втрачала всі завдання після перезавантаження сторінки.

### Причина
Service Worker для SQLite (`sqflite_sw.js`) не був підключений в `web/index.html`, через що IndexedDB не ініціалізувалась для персистентного збереження даних.

### Виправлення

#### 1. Файл: `web/index.html`

**Додано:**
```html
<script>
  // Register sqflite service worker for persistent IndexedDB storage
  if ('serviceWorker' in navigator) {
    window.addEventListener('load', function () {
      navigator.serviceWorker.register('sqflite_sw.js');
    });
  }
</script>
```

**Місце:** Перед закриваючим тегом `</head>`

### Архітектура збереження даних

#### Модель даних (Task)
```dart
Task {
  id: String
  title: String
  description: String?
  isCompleted: bool
  createdAt: DateTime
  dueDate: DateTime?
  completedAt: DateTime?
  priority: TaskPriority (none/low/medium/high)
  category: String?
  isArchived: bool
  isFocused: bool
}
```

#### База даних (SQLite)
- **Таблиця tasks:** зберігає всі завдання
- **Таблиця categories:** зберігає категорії
- **Зв'язки:** tasks.category_id → categories.id

### Примітки

- Service Worker працює лише на `https://` або `localhost`
- IndexedDB має обмеження розміру (50-100 МБ залежно від браузера)
- Дані можуть бути видалені при очищенні кешу браузера
- Для production рекомендується додати експорт/імпорт даних
