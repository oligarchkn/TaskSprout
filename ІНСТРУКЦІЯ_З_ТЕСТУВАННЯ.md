# Інструкція з тестування збереження даних

## Що було виправлено

**Проблема:** На веб-платформі завдання не зберігалися після перезавантаження сторінки.

**Рішення:** Підключено Service Worker для `sqflite_common_ffi_web`, який забезпечує персистентне збереження даних в IndexedDB браузера.

## Зміни в коді

1. **Файл:** `web/index.html`
   - Додано реєстрацію Service Worker `sqflite_sw.js`
   - Service Worker забезпечує роботу SQLite через WASM з збереженням в IndexedDB

## Де зберігаються дані

### Web (браузер)
- **Технологія:** IndexedDB (через Service Worker)
- **Локація:** DevTools → Application → IndexedDB → `tasksprout.db`
- **Персистентність:** Дані зберігаються постійно до очищення кешу браузера

### Desktop (Windows/Linux/macOS)
- **Технологія:** SQLite (файлова база даних)
- **Локація:** `%APPDATA%/com.example/task_sprout/databases/tasksprout.db`
- **Персистентність:** Дані зберігаються як звичайний файл

### Mobile (Android/iOS)
- **Технологія:** SQLite (через плагін sqflite)
- **Локація:** Внутрішнє сховище додатку
- **Персистентність:** Дані зберігаються до видалення додатку

## Перевірка IndexedDB в браузері

1. Відкрити **Developer Tools** (F12)
2. Перейти на вкладку **Application**
3. Розгорнути **IndexedDB**
4. Знайти базу `tasksprout.db`
5. Переглянути таблиці `tasks` та `categories`

## Очищення даних (для тестування)

### В браузері:
```javascript
// Відкрити Console в DevTools і виконати:
indexedDB.deleteDatabase('tasksprout.db');
location.reload();
```

### Або через UI:
1. DevTools → Application → Storage → Clear site data

## Troubleshooting

### Дані все ще не зберігаються:

1. **Перевірити, чи Service Worker активний:**
   - DevTools → Application → Service Workers
   - Має бути `sqflite_sw.js` зі статусом "activated"

2. **Перевірити консоль на помилки:**
   - Відкрити Console (F12)
   - Шукати помилки пов'язані з IndexedDB або SQLite

3. **Очистити кеш і перезавантажити:**
   - Ctrl + Shift + R (жорстке перезавантаження)
   - Або DevTools → Application → Clear storage

4. **Перевірити, чи браузер підтримує IndexedDB:**
   ```javascript
   // В консолі браузера:
   console.log('IndexedDB:', !!window.indexedDB);
   ```

5. **Переконатися що використовується HTTPS або localhost:**
   - Service Workers працюють лише на HTTPS або localhost

## Додаткова інформація

- Дані зберігаються локально і НЕ синхронізуються між пристроями
- Розмір бази даних обмежений квотою браузера (зазвичай 50-100 МБ)
- Дані можуть бути видалені браузером при нестачі місця (залежить від налаштувань)
- Для production рекомендується додати резервне копіювання/експорт даних
